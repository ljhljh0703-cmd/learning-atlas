# Apple 디자인 역분석 — 왜 좋아하나 · 어떻게 내게 적용하나 · 무엇을 더할까

## A. 페이지 해부 (iPhone 17 Pro에서 추출한 문법)
1. **풀블리드 제품 히어로**: 화면을 꽉 채운 한 장의 제품 사진(영상 루프) + 제품명만. 텍스트 최소.
2. **거대 중앙 선언문 + 짧은 부카피**: "Unibody 외장. 도약을 위한 탄탄한 기반." 한 화면 = 한 메시지. 헤드라인이 본문의 3~5배.
3. **스크롤 순차 공개(scroll storytelling)**: 섹션마다 스타트프레임→엔드프레임(스크롤로 장면 전환). 디자인→칩→카메라→성능→OS→AI 순서가 *영화 시퀀스*.
4. **하이라이트 갤러리**: 핵심 6개를 카드로 먼저 요약(디자인·A19·카메라·전면·iOS·AI) → 이후 각 항목 심층.
5. **대형 수치 블록**: "8배 광학줌 · 48MP · 31시간 · 50% 더 빠른". 숫자가 거대하게, 단위는 작게. **모든 수치에 각주[10]**(honesty device).
6. **라이트/다크 "방" 교차**: 밝은 섹션 ↔ 검은 드라마틱 섹션 번갈아 → 챕터 리듬·시선 환기.
7. **단일 액센트 = 제품색**: 코스믹 오렌지 등 *제품 자체가 유일한 색*. UI는 무채색. 색을 장식으로 안 씀.
8. **인터랙션은 "이해를 돕는" 것만**: 줌 단계 슬라이더, AR/3D 뷰어, 비교 토글. 장식 모션 아님 — 제품 이해 보조.
9. **여백이 주인공**: 요소 사이 거대한 숨. "복잡함은 사랑하되 복잡화는 싫다"(Vignelli와 동일 철학).
10. **SF Pro 단일 타입패밀리**, 큰 스케일 점프로 위계. 좌측/중앙 정렬, novelty 서체 0.

## B. 왜 사람들이(비전문가 포함) 애플 디자인을 좋아하나 — 심리
- **인지적 편안함(cognitive ease)**: 한 화면 한 메시지 → 뇌가 안 피곤. 이해가 쉬우면 "좋다"고 느낀다(처리 유창성=호감).
- **여백 = 자신감·고급의 신호**: 빈 공간을 살 수 있는 건 *자신 있는 브랜드*뿐. 여백이 곧 프리미엄 시그널(지위재 심리).
- **제품이 주인공(show, don't tell)**: 설명 대신 거대한 실물을 보여줘 *욕망*을 직접 자극. 글보다 이미지가 감정에 빠르다.
- **통제감·질서**: 픽셀 단위 정렬·일관성이 "이건 믿어도 된다"는 *신뢰*로 번역(디테일=품질 대리지표).
- **시퀀스의 쾌감**: 스크롤할 때마다 장면이 보상처럼 공개됨 → 도파민 루프(스토리텔링).
- **초점(한 번에 하나)**: 선택·정보 과부하 제거 → 결정 피로 0, "쉽다"는 체감.
- **감각적 디테일**: 부드러운 모션·반사·깊이가 *촉각적 만족*(sensory delight)을 줌.
- 요약: **명료(쉽다) + 여백(고급) + 실물(욕망) + 정밀(신뢰) + 시퀀스(쾌감)**. 비전문가는 이걸 "그냥 멋지다"로 느낀다.

## C. 역기획 — "나 = 제품"으로 적용 (포트폴리오 변환)
애플의 *제품 출시* 문법을 *사람 출시* 문법으로 치환. **이주형이 곧 iPhone**.
| 애플 섹션 | 나의 변환 |
|---|---|
| 풀블리드 제품 히어로 | 이름 + 1줄 포지셔닝을 풀스크린으로. 제품샷 자리 = 대표 작업 비주얼/3D 렌더 1컷 |
| 하이라이트 6 갤러리 | 내 핵심 역량 6개를 카드로 먼저 요약(하네스·게이트·지식그래프·추천·게임AI·디자인) |
| 디자인/카메라/성능 심층 | 대표 프로젝트 2~3개를 각각 *한 챕터*로: 거대 비주얼 + 한 줄 선언 + **각주 달린 실측 수치** |
| 대형 수치 블록 | "981 노드 · 88% 자동추출 · …" 거대하게 + 각주로 출처(=우리 evidence-label과 동형) |
| 라이트/다크 교차 | 프로젝트마다 톤 전환으로 챕터 리듬 |
| 비교(업그레이드할 만한가) | Before→After, 또는 "기획자 출신 → 시스템 빌더" 전환 서사 |
| 제품 사양 페이지 | = 이력서(상세 스펙) 링크 |
| 각주[10] | **정직성 장치 그대로 채택**: 모든 수치에 출처 각주 → 과장 없이 신뢰 |
- 핵심 규율: **한 화면 한 메시지 · 여백 사수 · 제품(작업물) 크게 · 수치엔 각주 · 색은 작업 비주얼에만**.

## D. 확장 — 애플 외 + 우리 시스템을 더 좋게
### D-1. 추가로 흡수할 가치 있는 레퍼런스 문법
- **Linear/Vercel(제품 정밀)**: 다크 네이티브·실토큰·반투명 보더(이미 흡수). 애플의 "스토리"에 이들의 "정밀 UI"를 결합.
- **에디토리얼/Kinfolk·잡지**: 큰 세리프·그리드·여백(우리 editorial-grid-canon). 애플 여백 철학의 출판 버전.
- **Awwwards 스크롤 시네마**: 스크롤 핀(sticky pin)·패럴랙스·로드 시퀀스. 애플 문법의 화려판.
- **NYT/데이터 저널리즘**: 스크롤리텔링 + 데이터 시각화(수치를 서사로). 내 "지식그래프" 서사에 적합.
- **3D/Behance(이미 템플릿화)**: 렌더 비주얼 = 애플의 "제품샷" 대체재.
### D-2. 우리 디자인 스위트를 더 좋게 (애플에서 배운 것 결선)
1. **"Apple-grammar" 아키타입 신설**: 스크롤 스토리텔링 + 한 화면 한 메시지 + 각주 수치 → 프리셋/스튜디오 용도에 "제품 출시형/포트폴리오 출시형" 추가.
2. **스크롤 핀 섹션을 toolkit에**: design-effects에 "sticky reveal/scroll-pin" 추가(현재 약함).
3. **각주-앵커드 수치를 honesty 표준으로**: evidence-label + 각주 = grade 하네스의 정직성 축 강화.
4. **여백·스케일 대비를 canon에 수치화**: 섹션 간 여백 토큰 상향, 헤드=본문 3~5×(애플은 더 과감) → DEVELOP-PLAN L1 보강.
5. **"한 화면 한 메시지" 린트**: 한 뷰포트에 메시지 1개 초과 시 경고(소비자 친화·인지편안함).
6. **제품(작업물)을 주인공으로**: 우리 미니랜딩 샘플도 텍스트보다 *비주얼 슬롯*을 키워 "show don't tell".
