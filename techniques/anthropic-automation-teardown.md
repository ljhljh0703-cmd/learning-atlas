# Anthropic 데이터 자동화 및 스킬 아키텍처 분석

> ✅ **확정(confirmed) 2026-06-14** — 세션 내 ③Gate 통과 + 작가 수용. (외부 미검증 세부는 본문 표기 유지.)

> **한줄**: 단순 모델 업그레이드가 아닌 표준 데이터셋(SSOT) 구축, TDD 검증, 그리고 데이터 코드와 스킬 문서의 CI/CD 동기화 강제를 통해 95%의 사내 데이터 분석 업무를 자동화한 사례 연구.
> **서브 브레인 핵심 질문**: 의사결정(decisions)이나 환경 스펙이 변경될 때 어떻게 AI 스킬과 mapping documents의 동기화 갱신을 강제하고, 오프라인 평가를 통해 성능 저하를 방지할 것인가?

---

## 1. Agentic Analytics Stack & 4단계 자동화 구조

### 1) 4층 레이어 아키텍처
*   **1층 (표준 데이터셋)**: 단 하나의 진실의 원천인 표준 데이터셋을 구축하고 메타데이터를 핵심 자산으로 관리하며, CI/CD 컨베이어 검수대를 통해 엄격한 규칙 준수를 물리적으로 강제합니다.
*   **2층 (진실의 원천)**: 공식 지표를 연산하는 **시멘틱 레이어(Semantic Layer)**, 데이터 계보(Lineage), 잘 큐레이션된 과거 쿼리 목록, 그리고 회사의 맥락 지식을 신뢰도 순으로 계층화하여 LLM에 주입합니다.
*   **3층 (스킬)**: AI에게 단순 가이드라인이 아닌 '절차적 지식(업무 매뉴얼)'을 모듈식으로 제공하는 레이어. 기존 21%에 그쳤던 데이터 쿼리 정확도를 95% 이상으로 도약시킨 코어 구조입니다.
*   **4층 (검증)**: 응답 신뢰도를 유지하기 위한 오프라인 평가 및 실시간 검증(Runtime Verification) 파이프라인.

### 2) 메트릭 유지보수 연쇄 동기화 로직
*   **지식의 낙후(Staleness) 극복**: 에이전트 도입 한 달 만에 정확도가 95%에서 65%로 무너지는 현상을 극복하기 위해, **에이전트가 참조하는 스킬 문서를 데이터 모델 코드와 동일한 Git 저장소(Repository)에 통합**했습니다.
*   코드 수정 과정에서 CI 훅(Hook)을 걸어 스킬 문서가 함께 갱신되지 않으면 빌드 및 커밋이 차단(Reject)되도록 강제하는 파이프라인을 가동합니다. 현재 전체 코드 수정의 90%가 스킬 문서 업데이트를 포함합니다.

## 2. 스킬(Skill) 디자인 및 Meta-Skills 설계

### 1) Trigger & Guard Logic
*   **지식 스킬 (Routing)**: 사용자의 질문 진입 시 AI가 전체 문서군을 무차별 검색하지 않고, 질문 영역에 가장 적합한 30개 남짓의 매뉴얼만 정밀 라우팅하는 스캔 필터를 구성합니다.
*   **가드레일 (Anti-pattern)**: 에이전트에게 질문이 들어오면 가장 먼저 '질문 명확화(Clarification)' 과정을 선행하도록 워크플로우를 고정(Constraint)합니다. 또한 주요 비즈니스 지표 정의는 에이전트가 독자적으로 판단하지 못하게 막고 사람이 관리하는 SSOT에서만 조회하도록 차단합니다.

### 2) Meta-Skills 템플릿
*   **오프라인 평가 자동화**: 에이전트(클로드)가 사내 대시보드를 직접 스캔하여 에이전트의 성능 저하를 방지할 **오프라인 모의시험 문제셋을 자동 생성**하는 메타-스킬(Skill Creator)을 설계했습니다.
*   과거 쿼리는 정제되지 않은 날것으로 주지 않고 리텐션 곡선, 퍼널 등 검증된 템플릿화된 분석 패턴으로 재가공하여 에이전트가 재사용할 수 있도록 설계합니다.

## 3. 심층 탐구 (Deep Inquiry)

*   **멘탈 모델 (Mental Model)**: "데이터 분석은 소프트웨어 코드가 아니므로 정답을 즉시 증명할 방법이 없다"는 전제. 따라서 정보의 양(Quantity)이 아닌, **질문을 데이터에 연결하는 '구조적 다리'**를 만드는 것에 주력하는 모델입니다.
*   **논리적 맹점 (Logical Blindspots)**: 낡은 문서와 stale한 정의는 에이전트 모델 자체의 지능 저하보다 에이전트의 실질 정확도를 더 빠르게 무너뜨리는 가장 취약한 논리적 맹점입니다.
*   **기회비용/임계점 (Trade-offs)**: 에이전트가 스스로의 답변을 의심하고 검증하는 '적대적 검토(Adversarial Review)' 단계를 추가할 경우, **정확도는 6% 상승하고 토큰 비용은 32% 절감되나, 응답 속도(Latency)는 72% 지연**되는 뚜렷한 트레이드오프 임계가 발생합니다.
*   **처참한 실패 시나리오**: 과거 성공 쿼리가 80% 이상 포함된 수천 개의 SQL 파일을 날것으로 에이전트에 직접 주입했음에도 정확도가 1%조차 상승하지 못했습니다. 구조화되지 않은 무차별 컨텍스트는 오히려 에이전트의 탐색 및 추론력을 완전히 마비시킵니다.

## 4. Sub-brain 자가 진화 설계 적용안

### 1) 스킬-코드 결합 TDD 린터 도입
*   의사결정 문서(`decisions.md`)나 core 환경 스펙이 변경될 때, 이와 의존성이 있는 에이전트 스킬(`.md`)이 함께 갱신되도록 git pre-commit hook 또는 `validate-skills` 단계에 **상호 갱신 강제 검수대(Knowledge TDD Linter)**를 결합합니다.

### 2) 오프라인 평가 기반 배포 차단
*   하네스에서 오프라인 모의 평가 문제셋(Evals)을 자동 가동하여, 주요 기능의 정답률이 90% 미만으로 떨어지는 패치가 발생하면 staging pull request의 정본 머지(merge)를 원천 차단합니다.

### 3) 자가 진화 피드백 루프
*   에이전트 응답 하단에 **출처 푸터(Metadata 및 신선도 타임스탬프)**를 강제 부착하고, 별도의 audit 에이전트가 채널 및 로그를 순회하며 인간의 오류 교정 지적을 자동 스캔하여 `Memory.md`에 `[Rule Candidate]`로 병합 제안하는 체계를 확립합니다.
