# CoC 시나리오 해부 v2 — 직접 해체 완료

> **학습 규약 (작가 정정 2026-07-17)**: 원문 표현·문장·고유 텍스트는 **온전히 학습한다** — 그래야 안 베낀다(어설픈 우회 요약이 오히려 누수). **복제 금지 게이트 = 산출 쪽**: 게임 텍스트(Pack 콘텐츠) 생산 시 원문 표현 재사용 0 — Penelope/Codex 콘텐츠 게이트에 원문 대조 검사로 걸 것. vault 노트는 기존 컨벤션 유지(구조·분석 자유, 원문 대량 전사는 안 함).
> **기존 자산 (바퀴 재발명 방지)**: index 이하에 이미 — 두 PDF의 인덱스 노트(첫-번째-어둠·2019-수상작-모음) + **작가 자작·운영 노트 8편**(그하숙집·극락정토·기억의공백·조선크툴루상·머리검은짐승·서울한밤·오사이비자·황금빛찻잔 — 전부 작가가 키퍼로 실제 운영!) + 설정집(심해인 등). 본 노트 = 그 위의 **구조 해부 층**(파이프라인 관점) — 개별 시나리오 내용은 운영 노트가 정본.

## 2. 해부 스켈레톤 v2 (12요소 — v1의 9요소 + 신규 3)

1. **배경·진실 (키퍼 전용)** — 실제로 일어난 일의 역사. 플레이어 정보와 엄격 분리.
2. **수호자 참고사항 (서두!)** — 난이도·개변 자유("결말을 따르지 않아도 됨")·캠페인 접속 훅. **개변 허용이 명문화된 장르** — 키퍼 재량은 버그가 아니라 스펙.
3. **타임라인 이중 구조** ★신규 — "사건의 경과"(과거 연표: 날짜별 사실) + **"앞으로의 사건"(탐사자가 개입 안 하면 진행되는 미래 — 힘의 발현이 점점 잦아짐)**. 후자 = **우리 신화 틱/Agenda의 TRPG 원형.**
4. **훅(도입)** — 일상 묘사 → 침입 사건(초인종·의뢰·목격). "무엇을 하고 있었는지 물어보라" = 일상 정착 후 파열.
5. **등장인물 = 중심 자산** — 서사 역할 + 관계 + **롤플레이 포인트**(연기 지침: 말투·태도·반응 트리거) + 스탯 블록은 부록 분리. **가변 진범의 이중 서술** ★신규: 용의자 NPC마다 "무죄 버전 / 범인 버전" 병기 — 진범은 키퍼가 고름.
6. **사실과 단서** — NPC별 제공 정보 불릿·문서/물건 = 명명 오브젝트(접촉 트리거: "만지면 판정"). **축적형 결말** ★신규: "어느 하나의 결정적 단서가 아니라 행동·발견의 총합이 결과를 결정" — 우리 evidence 카운터+게이트의 원형.
7. **장소 절** — 장소마다 묘사+단서+조우. 지도 동반.
8. **행동 갈래** — "탐사자들이 취할 만한 행동" 목록 + 행동별 결과 절(자유 입력을 유한 절로 수렴) — **우리 고정 선택지·조우 카드의 원형.**
9. **힘의 발현(점증 위협)** — 약한 징조→심각. 파멸 밴드 사다리와 동형.
10. **페이싱 복구 장치** — "샛길에 오래 머무르면 (핵심 사건)을 강조하라" — 수동 신화 틱.
11. **핸드아웃 부록** — 시나리오별 네임스페이스("핸드아웃 서울 1") + 권말 모음 인쇄(플레이어에게 물리적으로 건네는 명명 단서).
12. **결말·정산** — 엔딩 2~4(열린 결말 허용) + 이성 회수/보상 표.

## 3. 파이프라인 매핑 v2 (엘드리치 서울 계약 대응)

| CoC 요소 | 우리 계약 | 변환 노트 |
|---|---|---|
| 키퍼 전용 진실 | hidden visibility | 동일 문법 |
| **미래 타임라인** | **신화 틱·응보 게이지** | 방치 시 진행하는 위협 = 이미 설계됨(선례 확인) |
| 롤플레이 포인트 | NPC·동료 `voice`/대사 지침 필드 | 데이터 필드로 승격 |
| 가변 진범 이중 서술 | **사전 저작 분기**(진범 A/B별 guard·단서 스왑) | "키퍼 재량"의 결정론화 — 가장 무거운 변환. Penelope가 이중 서술을 분기 데이터로 컴파일 |
| 축적형 결말 | evidence 카운터 + route+gate | 동형 — 우리가 이미 맞는 설계 |
| 행동 갈래 | 조우 카드 approaches | 동형 |
| 핸드아웃 | 명명 단서(일지 수집) | 시나리오별 네임스페이스 채용 |
| 개변 자유·튜닝 조언 | 튜닝 재량+주석 계약 | 동형 |
| 대인 기능 중심 유형 | 연결 레인 | 유형→레인 매핑 |

**시나리오 유형 3종 관찰**: 인물 중심(꿈의 소집 — NPC 비밀·욕망이 콘텐츠) / 장소 중심(하숙집·극락정토 — 공간 탐사) / 액션 중심(룰북 2편·추격). → ScenarioSource에 `유형` 필드 — 유형이 콘텐츠 배분(NPC 수 vs 장소 수 vs 조우 수)을 결정.

## 5. 산출 게이트 (클린룸의 올바른 위치)
Pack 콘텐츠 생산 시(M0.7·Penelope): 학습 코퍼스 원문과의 **연속 일치 검사**(n-gram 대조)를 콘텐츠 게이트에 추가 — 구조는 배우고 표현은 새로 쓴다. (검사 대상 = 산출물이지 학습이 아님.)
