# webnovel-writer 해체 — 장편 서사 시스템의 골수 6종

> 정체: Claude Code 기반 장편 웹소설 창작 시스템(200만 자급 연재의 망각·환각 해결이 목표). **비슷한 프로젝트(장편 서사·시나리오 엔진·캠페인 시스템) 착수 시 이 노트부터** — 구조 문제의 상당수가 여기 이미 풀려 있다.

## 골수 1 — Story System: 계약 → 커밋 → 투영
```
계약 시드(MASTER_SETTING + 권 계약 + 반패턴 설정)
 → 런타임 계약 생성 + 쓰기 전 검증(preflight)
 → 장(章) 커밋 수락(CHAPTER_COMMIT: accepted_events / state_deltas / entity_deltas / summary)
 → projection writers가 state·index·요약·장기기억을 투영(read-model)으로 갱신
 → 이벤트 감사 체인(events + 수정 제안 + 덮어쓰기 장부)
```
- **진실원과 투영의 엄격 분리**: `.story-system/` = 주쇄 진실원, 파생 상태 전부 = 투영(재계산 가능). 우리 event truth·display 분리·Pack 계약과 동형 — **독립 수렴 = 구조 자신감**. Penelope 스토리 상태 관리의 직접 참고.

## 골수 2 — 방환각 3정률 (한 줄 공식들)
| 정률 | 공식 | 집행 |
|---|---|---|
| 대강 = 법률 | 대강을 벗어난 자유 발휘 금지 | 쓰기 전 컨텍스트 에이전트가 장 대강 강제 로드 |
| 설정 = 물리 | 설정 자기모순 금지 | 리뷰어의 일관성 검사(전력·장소·타임라인) |
| **발명은 등록** | 새 엔티티는 반드시 추출·중복 해소 후 입고 | 데이터 에이전트 자동 추출 |
→ **"발명은 등록"** = 확장 AI의 핍진성 가드레일 집행 메커니즘(Penelope 계약 ②의 구현 형태). 발명 금지가 아니라 발명의 **회계 처리**.

## 골수 3 — Strand Weave 리듬 (수치화된 직조 레드라인)
주선(Quest) 60% / 감정선(Fire) 20% / 세계관(Constellation) 20% + **단절 상한**: 주선 연속 ≤5장 · 감정선 공백 ≤10장 · 세계관 공백 ≤15장.
→ 직조 비율(우리 7:3)의 장편판 — "무엇을 섞나"뿐 아니라 **"얼마나 오래 끊겨도 되나"**를 수치로. 캠페인·시즌 설계 시 이 3선 상한 문법 채용.

## 골수 4 — 추독력(독자 견인) 체계
- **훅 5형**: 위기·수수께끼(悬念)·갈망·정서·선택 — [게임 속 바바리안으로 살아남기 — 장편 문법 해부 v1 (구조 스캔)](barbarian-webnovel-longform.md) 실측 5형과 독립 수렴(**교차 검증 완료**).
- **운용 규칙**: 장당 기대 앵커 1 + 미세 회수(微兑现) 1~2 · 동일 훅 3연속 금지 · 재사용 시 대상/대가/결과 중 1요소 차별화 의무 · 기대 부채 경고(회수 안 하면 늑대 소년 효과, 너무 쉬우면 무장력).
- **쿨포인트 3막 구조**: 포석 30% → 회수 40% → 여파 30%(반응·수확·새 기대).
- **장르별 훅·회수 라이브러리** — 크툴루 편 내장: 훅 = "깨진 규칙의 반전"("깨진 규칙이 거울 방에선 반대로 작동")·봉인 조기 붕괴·구조 vs 의식 차단 양자택일·이성 임계+기억 결손 / 회수 = 단서 봉합(3장 전 지도 표기가 오염원과 일치)·**"실행 가능한 생존 규칙 1개 확인"**·공포 속 능동 결단·의심→공동 부담 관계 전환.
→ 게임 번역: 조우 카드 말미 훅(5형 로테이션) + run 단위 기대 부채 관리 + 명명 단서 = 미세 회수.

## 골수 5 — 6차원 리뷰어 (콘텐츠 검수 골격)
쿨포인트 밀도 / 설정 일관성 / 페이싱(Strand 비율·단절) / OOC(인물 이탈) / 연속성(장면·서사) / 독자 견인(훅 강도·기대 관리). → E3 검수 체크리스트의 완성 골격(+우리 고유: 톤 금칙 3.5~3.7·n-gram 원문 대조·15세 수위).

## 골수 6 — deconstruction-agent 패턴
참고작 해체 전용 서브에이전트: "이전 가능한 창작 패턴과 초기화 후보로 해체하되 **원작 사실을 복제하지 않는다**(신작 canon 오염 방지)". → 우리 레퍼런스 학습(CoC·바바리안 해체)의 에이전트화 — 파이프라인 단초 인터뷰 단계의 자동화 후보.

## 불채택 (근거 실명)
- **RAG 스택**(임베딩+리랭커 API 필수) — vault grep>벡터·infra-0 규율. 구조(계약·커밋·투영)는 RAG 없이 성립.
- **코드 이식** — GPL-3.0 전파. 패턴·스키마·수치만.
- 장 단위 최적화의 1:1 이식 — 게임은 씬·조우·run 단위로 번역 필요.
- archive 사양 다수 = 버전 간 개념 드리프트 주의(overview.md만 최신 정본).
