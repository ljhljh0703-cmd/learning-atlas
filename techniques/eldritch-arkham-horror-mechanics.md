# 엘드리치 호러 · 아컴호러 카드게임 — 메커니즘 레퍼런스

> ⚠️ 임시(provisional). 용도: eldritch-seoul-rpg-pre-gdd 코어 루프 설계 레퍼런스. 작가 발화(2026-07-17): "플레이어는 엘드리치 호러·아컴 호러처럼 캐릭터를 육성하면서 시나리오를 진행. 위기 대응, 큰 위험=큰 보상, 일정 턴에는 위협(신화 단계, 응보 등). 동료는 AI가 보조하는 느낌 — 당장은 입력된 대로 행동하는 NPC."

## 1. 엘드리치 호러 (FFG 2013, 보드게임)

**라운드 3단 구조** — ① 행동(이동·휴식·자산획득·티켓·집중 등 2행동, 동일 행동 중복 금지) ② 조우(위치 기반 카드: 지역/조사/차원문/탐험 — 초기효과→성공/실패 분기 3단 텍스트) ③ **신화 단계**(신화 덱 1장 강제 공개 = 시스템이 조사자를 때리는 턴).

**압박 기계 (신화 단계의 부품)**:
- **징조 트랙**: 원형 4칸을 신화 카드가 전진시키고, 징조와 같은 무늬의 차원문 수만큼 **파멸 트랙 전진** — 방치한 위협이 이자로 돌아오는 구조.
- **파멸 트랙(doom clock)**: 0 도달 = 고대의 존재 각성(패배 임박·2페이즈 돌입). 전역 타이머.
- **응보(Reckoning) 효과**: 신화 카드가 트리거하면 응보 아이콘 가진 모든 카드(괴물·소지품·상태)가 일제 정산 — 빌린 힘(부채·저주·어둠의 계약)의 청구서.
- **양면 카드**: 마법·상태 카드 뒷면 확인 금지 — 응보 시 뒤집힘. *힘을 얻는 순간 대가를 모른다* = 코스믹 호러의 기계화.
- **미스터리 3장 + 최후의 미스터리**: 승리 = 상태 축적이 아니라 저작된 목표 카드 순차 해결.

**판정**: 스탯 개수만큼 d6, 5-6 성공. 수정치는 주사위 증감, 최소 1개 보장. 보너스 자산 1개만 적용(스택 방지), 리롤 자원(단서·집중)은 별도 축. 축복/저주 = 성공 임계 이동(4+/6).

**전투 = 2연속 테스트**(의지→공포 피해, 힘→체력 피해·괴물 피해 동시) — 회피/도주 없이 앞으로 넘어지는 간결화.

**캠페인 이월(니알라토텝의 가면들)**: 고대의 존재 격파 시 자산 초기화, 단 **어둠의 계약·권능의 약속은 이월** — 대가만 남는 이월. 개인 이야기 성공/실패도 이월.

## 2. 아컴호러 카드게임 (FFG 2016, LCG)

**라운드 4단**: **신화**(파멸 토큰 +1 → 임계 도달 시 주요사건 진행 + 전원 조우 카드 1장) → **조사**(1인 3행동: 조사/이동/공격/회피/카드 플레이…) → **적**(사냥꾼 이동·교전 적 공격) → **정리**(드로우+자원).

**이중 덱 시계 — 이 게임의 심장**:
- **주요목적(Act)**: 조사자가 단서를 모아 *능동적으로* 진행시키는 이야기 전진.
- **주요사건(Agenda)**: 파멸 토큰이 쌓이면 *자동으로* 진행되는 위협 전진(항상 불이익).
- ∴ 같은 시간축 위에서 "내 이야기가 먼저 도착하나, 신화가 먼저 도착하나"의 경주. 모든 턴에 시간 압박이 걸림.

**판정 = 혼돈 주머니**: 능력값 + 소모(commit) 카드 보정 + 혼돈 토큰 ≥ 난이도. 토큰에 자동실패(촉수)·고대 표식(조사자별 특수효과) 존재. 손패를 판정에 태우는 commit = *자원을 태워 위기 대응*.

**캠페인 구조**: 시나리오 종료 → 결말 분기(→결1 식) → **캠페인 로그에 기록·다음 시나리오가 참조**(선택의 지연 회신) · 승점=XP → 덱 향상(육성) · 쓰러지면 **트라우마**(체력/정신 한계 영구 잠식, 한계 도달=조사자 사망/발광). 육성과 흉터가 같은 저장소에 쌓임.

**동료(Ally) = 슬롯 자산**: 손2·동료1·신체·액세서리·마법2 슬롯 중 동료 슬롯에 놓는 카드. **카드에 적힌 효과대로만 작동**(스탯 보정·기능 격발·피해 대신 받기), 자체 체력/정신력 보유 = 소모 가능한 방패. *지능 없는 scripted 보조* — 작가가 말한 "입력된 대로 행동하는 동료 NPC"의 검증된 원형.

## 3. 엘드리치 서울 시사점 (매핑)

| 레퍼런스 부품 | 엘드리치 서울 대응 | 상태 |
|---|---|---|
| 신화 단계(강제 위협 턴) | **부재** — 현재 매 턴 = 플레이어 choice만. 일정 턴마다 신화 단계 삽입 = 신규 설계 필요 | 🔲 기획 |
| 파멸 트랙 | `causality`(인과율 예산, doom clock)가 이미 동형 — 신화 단계의 전진 대상으로 연결 | ✅ 접합 |
| 응보(Reckoning) | 부재 — "빌린 힘의 청구서" 축(confront 피해 보상·부채류 상태와 접합 후보) | 🔲 기획 |
| Act/Agenda 이중 시계 | spine 진행(6 stage) = Act 등가. Agenda(자동 전진 위협) = 부재 — 신화 단계와 한 몸으로 설계 | 🔲 기획 |
| 판정+commit 자원 | check(stat+resource_bonuses) 기존 계약 + 리롤/부스트 자원 소모 = 위기대응 감각 | ✅ 부분 |
| XP→덱/스탯 향상 | 육성 = stats 게이트(route+gate). run 내 성장 + unlock-only 메타 | ✅ 있음 |
| 트라우마 이월 | 부재 — unlock-only 원칙과 충돌 여지(부정 이월). 후속 결정 | 🅿 보류 |
| 동료 Ally 슬롯 | **scripted 동료 NPC 모델** — 결정론 효과표(스탯 보정·격발 기능·피해 대신 받기·이탈 조건). belief/relation 엔진 불요 = 안배 조항과 무충돌 | 🔲 기획(우선) |
| 양면 카드(대가 은닉) | 서술 층에서 재현 가능(획득 시 효과만 공개, 응보 시 대가 공개) — 결정론 유지 | 💡 후보 |

**취향 신호(보드 갱신 후보)**: ① 위협이 *일정에 따라 자동으로 오는* 구조(플레이어 입력과 무관한 시계) ② 큰 리스크=큰 보상 + 청구서 나중 도착 ③ 동료 = 소모 가능한 scripted 자산 ④ 육성이 결말 자격을 산다. → game-design-discovery §1 취향 보드 반영은 후속(파일 미확인).
