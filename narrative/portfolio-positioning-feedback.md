# 포트폴리오 포지셔닝 피드백 (멋사 멘토)

> 작가가 **AI로 HTML 포폴 만들기 이전** 옛 이력서에 받은 피드백 + Q&A. 포폴/이력서 작업 시 *맥락·방향성·어필법* 앵커. 문장 레벨 규칙은 juhyeong-voice §6(준격식 프로파일)과 한 쌍.
>
> 캐빗: 피드백(1번 Notion)은 Jina Reader 경유 충실 distill. **Q&A(2번 Notion)는 작가 제공 verbatim** — 아래 프로젝트 실명·❌/✅ 예시·서술 3축은 원문 그대로.

## 어필 대원칙 (한 줄) 🎯

> **1순위 타깃 = AI 프로덕트/엔지니어** (2026-07-02 확정). 아래 대원칙·앵커는 이 타깃 기준. 게임사 지원 시 [게임 기획자 포폴·취업 지식 베이스 (유리링)](../methods/game-planner-portfolio-guide.md) 병용.

**"기술을 구현했다"가 아니라 "왜 이 기술을 선택했고, 어떤 비즈니스 문제를 해결했는가."**
17억 매출 PM 배경 + LLM·RAG·파인튜닝 *직접* = 순수 개발자 지망생과 갈리는 지점. 이 **비즈니스-기술 하이브리드** 관점이 프로젝트 서술 *한 줄 한 줄* 에서 배어나야 함. → positioning-domain-specific-harness 와 정합.

## 왜 "전달"이 문제인가 (핵심 진단)

채용담당자 리뷰 순서: **기본정보 → 간단소개 → 경력·프로젝트 스캔 → (흥미 시) 포폴 클릭.**
→ **이력서 본문에 프로젝트 기술서가 없으면 포폴 클릭 단계까지 못 감.** 콘텐츠는 강한데 *전달 구조* 가 그걸 가림. 작가처럼 부트캠프 3개 + AI 에이전트 6종 + 보드게임 2910 자산이 있는데 본문에서 안 보이면 큰 손실.

## 방향성 원칙 (do's)

1. **전달 구조 = 설계적 사고의 증거.** 기획/PM 포지셔닝이면 이력서 구조 자체가 설계력을 말한다. 대기업 표준 템플릿 → 테크회사용.
2. **헤드라인 뒤 3~5문장 소개 필수.** 경력 다 읽기 전에 프로필이 잡혀야.
3. **역량을 포폴 열기 *전에* 가시화.** AI 프로젝트를 교육 섹션에 묻지 말고 이력서 본문에 기술 스펙 노출.
4. **접근 마찰 0.** 구글드라이브 폴더(위계 불명·인지부하·권한에러) → Notion 공개 페이지 + 프로젝트별 GitHub 링크.
5. **다 나열 말고 2~3개 깊게.** 선별 기준 = ①타깃 직무 연관성 ②본인 기여도 ③기술적 판단 경험 有.

## 프로젝트 서술 표준 양식

```
프로젝트명 — 한 줄 설명 | 기간 | N인 팀 / 역할
기술 스택: [실제 사용 기술만]
- [왜 이 기술·방식을 선택했는가]
- [내가 직접 담당한 것]
- [결과가 어떻게 달라졌는가 — 수치 또는 변화 방향]
```

## 서술 3축 — "단순 구현자 vs 판단하는 기획자"를 가르는 지점

**축 1 — 내가 직접 *결정* 한 것.** 팀이 한 일 X, 내가 판단·결정한 일 O. 작가 사례:
- "RAG 추천에서 단순 벡터 검색 아닌 **Dense+BM25 하이브리드** 선택 이유"
- "17개 RDB 테이블 중 **절반을 서비스 페르소나 로직에 할당**한 판단"
- "의료 챗봇 **RedFlag 필터를 기획자가 직접 설계** — 모델 정확도보다 '정서적 안전선'이 우선이라는 판단"

**축 2 — 기술↔비즈니스 연결.** 기술 선택에 "왜 이게 비즈니스 목표에 맞나" 한 줄:
- ❌ "HyperCLOVA X + Qdrant 기반 RAG 파이프라인 구현"
- ✅ "낮은 리텐션 문제 해결 위해 HyperCLOVA X + Qdrant 하이브리드 검색 설계 — **정확도 향상보다 '다시 돌아오게 하는 구조' 설계에 집중**"

**축 3 — 결과 or 변화 방향.** 수치 있으면 수치, 없으면 방향:
- 수치: "제안서 작성 5일 → 2일 단축"
- 방향: "수동 법률 검토 → AI 에이전트 자동화로 검토 리드타임 대폭 단축"

## 작가 프로젝트 — 우선순위 + 즉시 적용 어필 라인 (멘토 초안)

이력서 본문에 담을 **핵심 2~3개** 우선순위:

**1. 북켓몬 (AI 도서 큐레이션)** — RAG 파이프라인 설계 + 게이미피케이션 기획 + 팀장 리드. *AI 기술 + 서비스 기획 + 협업* 모두 드러남.
> 강조: 리텐션 높이는 *구조* 설계. "단순 책 추천이 아닌, 다마고찌·포켓몬 구조의 AI 독서 파트너 기획 → 경험치·진화 시스템으로 'AI 파트너와 함께 성장하는 감각' 설계" → *구현자 아닌 설계하는 기획자* 인상.

**2. 의료 자문 챗봇** — LoRA 파인튜닝 + RedFlag 안전 필터 + UX 라이팅. *기술 이해도 + 기획자 역할*.
> 강조: "의료 AI에서 정확도 외 '사용자가 안전하다 느끼는가'가 리텐션·신뢰를 결정한다 판단 → RedFlag 필터·UX 라이팅을 기획자 역할로 직접 설계."

**3. AI 에이전트 6종 (또는 보드게임 2910)** — 자동화 파이프라인 구축.
> 강조: "게임 기획 검증·라이브 운영·법률 검토 등 6개 도메인 에이전트 직접 설계·구현 — **'나만 쓰는 도구'가 아닌 '팀의 의사결정 사이클을 단축하는 공용 인프라'** 로 설계."

## 학습→반영 루프 (반영처)

- **[게임 기획자 포폴·취업 지식 베이스 (유리링)](../methods/game-planner-portfolio-guide.md)** — 게임 기획자용 도메인 지식 짝(역기획서·7대죄악·포지션별 주제·1분자소·면접). 본 노트 = 일반 전달·포지셔닝, 저 노트 = 게임 기획 특화 무엇을/양식. 게임사 지원 시 함께 로드.
- **juhyeong-voice §6 준격식 프로파일** — 문장 레벨 규칙(명료성·번역투 박멸). 본 노트 = 구조·포지셔닝·어필 레벨. 한 쌍.
- **bespoke-html-direction** — HTML 포폴 제작 시 구조 원칙(3~5문장 소개·프로젝트 2~3개 깊게·GitHub 링크·접근마찰0) + 서술 3축을 섹션 레이아웃에 배선.
- **`applications/<회사>/strategy.md`** — 회사별 지원 시 포지셔닝 앵커(비즈니스-기술 하이브리드) 재진술 + 타깃 직무 연관 프로젝트로 2~3개 재선별.
