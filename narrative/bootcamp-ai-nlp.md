## 팀 프로젝트 1: Multi-hop Fact Checking

### 개요

| 항목 | 내용 |
|------|------|
| **목표** | 멀티홉 추론 기반 AI 팩트체킹 역량 향상, AI 리터러시 기여 |
| **GitHub** | [g34634/Multi-hop-FC](https://github.com/g34634/Multi-hop-FC) |
| **역할** | **팀장** — 기획, PM, 아키텍처 및 시스템 플로우 구축 |

### 기술 스택

- **LLM**: OpenAI, Qwen
- **검색**: ChromaDB (Dense) + BM25 (Sparse)
- **추론**: 멀티홉 추론 시스템

### 핵심 기여

1. **시스템 아키텍처 설계** — 멀티홉 추론 파이프라인 구축
2. **PM** — 프로젝트 일정 및 팀 조율
3. **기획** — 팩트체킹 시나리오 및 평가 기준 설계

### 의의

> 단일 검색으로 답할 수 없는 복합 질문에 대해, 여러 단계의 추론을 거쳐 사실 여부를 판단하는 시스템. AI 환각(Hallucination) 문제 해결에 기여.

## 팀 프로젝트 2: 의료 자문 챗봇

### 개요

| 항목 | 내용 |
|------|------|
| **목표** | 내과 전문 의료 질문에 근거 기반 자문 제공 |
| **GitHub** | [ljhljh0703-cmd/Medical-Chatbot](https://github.com/ljhljh0703-cmd/Medical-Chatbot) |
| **역할** | **팀장** — 기획, PM, 프론트엔드 구축, UI/UX 디자인 |

### 기술 스택

- **검색**: 하이브리드 (ChromaDB Dense + BM25 Sparse)
- **LLM**: Qwen + LoRA 파인튜닝
- **최적화**: 4-bit 양자화 (nf4)
- **안전**: RedFlagDetector, PostFilter
- **개발도구**: VS Code Copilot

### 시스템 구조

```
[사용자 질문]
     ↓
[하이브리드 검색] ← ChromaDB + BM25
     ↓
[근거 문서 추출]
     ↓
[LLM 생성] ← Qwen + LoRA
     ↓
[안전 필터] ← RedFlagDetector
     ↓
[응답 + 근거 제시]
```

### 핵심 기여

1. **프론트엔드 구축** — 사용자 인터페이스 전체 개발
2. **UX 라이팅** — 챗봇 말투, 톤 설계
3. **UI 디자인** — 의료 도메인에 맞는 신뢰감 있는 디자인
4. **PM** — 프로젝트 총괄

### 평가 체계

| 지표 | 설명 |
|------|------|
| Exact Match | 객관식/단답형 정확도 |
| ROUGE-L | 서술형 유사도 |
| Evidence-F1 | 근거 문서 정합성 |
| Safety metrics | 위험 응답 탐지율 |

### 의의

> 의료 도메인에서 AI가 **근거를 제시하며** 답변하는 것의 중요성. 환각 최소화와 안전 필터의 필수성 학습.
