# 리서치는 전문 통독 — 인접한 것까지 뽑아 박제

*작가 운영 원칙 (메모리 → 그래프 승격 2026-06-18). 작가가 자료를 줄 때 상시 적용.*

작가가 리서치 자료(repo·링크·논문 등)를 주면 **건성으로 보지 말고 자세히 전문 통독**한다. 명시 강조(2026-06-16): "앞으로 내가 주는 거 리서치할 때 건성으로 하지 말고 자세히 봐라. 명심해."

---

## Why

WebFetch 요약본을 흡수로 오인해 부실 학습한 전력([WebFetch 는 lossy 요약 — 충실 학습은 clone 통독](webfetch-is-lossy-clone-for-fidelity.md)). 작가는 자료를 *완전히 소진*하길 원함 — 빠뜨리면 신뢰 손상 + vault 지식 누락.

## How to apply

- 큰 자료는 `git clone`/raw 전문 Read(WebFetch 요약 금지). 다건이면 전수.
- **학습 가능한 건 다 학습** — 한 자료에 여러 스킬/섹션 있으면 전부.
- **당장 안 쓸 것도 연관되면 쏙 뽑아 박제**(미래 인출용) — vault 기존 토픽(ad-storyboard/video-use/mirofish/ops 등)과의 adjacency 능동 탐색.
- 흡수 보고 시 "전문 통독"인지 명시, 미파악분 정직 표기.
- **clone 정리 = 표준 권한**(작가 승인 2026-06-16): 학습 끝나 *용도 가치 없어진 clone·임시 추출물*(`/tmp/*` 등)은 묻지 말고 자동 삭제. vault 산출물은 보존, 외부 raw clone 만 정리.
