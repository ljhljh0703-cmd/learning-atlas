# 토스 앱인토스 — 서비스 오픈 정책

> 출처: https://developers-apps-in-toss.toss.im/intro/guide.md
> 학습 동기: 출시 차단의 *구체* 룰. 특히 §5 생성형 AI 정책 = NPC harness welfare-gate / 작가성 윤리와 직접 동형.

## 한 줄 요약

**6 정책: 금지 콘텐츠 / 인허가 카테고리 / 어뷰징 방지 / 외부 링크 차단 / 생성형 AI 고지·표시 / 로그인·결제·광고 단일화.** 사용자 보호 + 플랫폼 일관성이 모든 룰의 두 축. 생성형 AI 위반 = 과태료 3,000만원.

---

## 1. 앱인토스 직접 적용 (런칭 시 참고)

### §1 출시 불가 카테고리 8종
1. 디지털/가상자산 (NFT 포함)
2. 자금세탁 가능 (현금 환전/환불)
3. 불법 조장 (해킹/신분 조작)
4. 사행성/베팅
5. 금융 상품 중개·판매·광고
6. 투자 자문/리딩방
7. 의료 (예외: 공공데이터 단순 조회만)
8. 내부 정책상 승인 불가

### §2 인허가 필요 카테고리
- 의료 — 공공데이터 + 순위/예약 X + 광고비 X
- 쇼핑몰 — 고객센터/가품/환불 정책 필수
- 교육 — 국가 자격증 없는 수익 X

### §3 어뷰징 방지
- 동일 워크스페이스 내 같은 핵심 기능 미니앱 다중 출시 X (테마/주제만 다른 것은 동일 앱)
- 동일 브랜드 다중 미니앱 X (목적·기능 명확 다른 경우만 별도 허용)

### §4 자사앱 설치 / 외부 링크
- 자사 앱 설치 유도 *전부 금지* (문구/배너/링크/혜택)
- 외부 링크 — 법률 고지 / 공공기관 / 단순 정보 / 제휴 결제 외 차단
- "앱 내 기능은 미니앱 내 완결"

### §5 ★ 생성형 AI 정책
- **사전 고지** — 첫 이용 / 첫 AI 기능 사용 시점
- **표시 의무** — AI 결과물에 라벨/배지/워터마크
- **위반 시:** 과태료 ≤ 3,000만원 / 자료 제출 요구 / 현장 조사 / 서비스 중지

### §6 단일화 정책
| 영역 | 허용 |
|------|------|
| 로그인 | 토스 로그인 only |
| 실물 결제 | 토스페이 only |
| 디지털 결제 | 인앱결제 only |
| 광고 | 앱인토스 전면형/보상형/배너 only |

## 2. 응용 — 일반 패턴 (Sub brain/NPC harness 차용)

### ★ 패턴 1: 출시 불가 카테고리 = 명시적 negative space
"무엇이 가능한가" 가 아닌 "무엇이 불가능한가" 8개 + 8번째 = "그 외 정책 위반". **negative space 명시 + open clause 1개.**

→ NPC harness 응용:
- 작가성 룰을 *허용 list* 가 아닌 *금지 list* 로 작성. open clause "그 외 welfare 침해 우려" 1개.
- BEHAVIOR.md anti-pattern 섹션이 이미 부분 적용. 강화 가치.

### ★ 패턴 2: "법적 인허가 무관" 절단
대출/보험 등 *합법이어도 차단*. 법률 ≠ 플랫폼 기준. **플랫폼 기준이 더 엄격할 수 있음** 을 명시.

→ NPC harness 응용:
- "AI welfare 가 법적 의무는 아니지만 우리는 강제" — welfare-gate.md framing.
- 토스 패턴이 우리 입장의 정당화 근거.

### ★ 패턴 3: 어뷰징 = "검색 점유 위한 다중 출시"
*노출 점유* 자체를 anti-pattern 으로 지정. 양은 품질이 아님.

→ NPC harness 응용:
- 작가가 같은 캐릭터 변종 다수 등록 → 어뷰징. 캐릭터 1개 깊이 우선.
- npc-eval 룰: 캐릭터 유사도 검출 (persona-vector cosine) 적용 검토.

### ★ 패턴 4: "완결성" 강제 — 외부 링크 금지
앱 내 기능은 *미니앱 내 완결*. 외부 의존 = 사용자 신뢰/보안 risk.

→ NPC harness 응용:
- NPC 응답이 *대화 내 완결* — "자세한 건 작가에게 물어봐" 같은 회피 금지.
- BEHAVIOR.md anti-pattern: "메타 회피" 추가 가치.

### ★★★ 패턴 5: 생성형 AI 사전 고지 + 표시 의무
**우리 NPC 게임에 직접 적용.** 토스 룰 = 한국 법령 반영.

→ NPC 게임 적용 (M1 출시 전 강제):
- *첫 플레이 onboarding* 에 "이 NPC 는 AI 가 응답합니다" 고지
- NPC 응답에 *시각적 표지* (말풍선 색/아이콘)
- 위반 시 과태료 3,000만원 → 게임 출시 전 법률 검토 필수

→ welfare-gate.md / 작가성 윤리 3축에 *법적 의무 axis* 추가.

### ★ 패턴 6: 단일화 = 다양성 절단의 정당화
로그인 1개 / 결제 2개 / 광고 1군 — *다양성 흡수 X, 강제 단일*. 토스 해상도/SDK 단일과 동형, 더 강함.

→ NPC harness:
- 메모리 schema 1개 / voice spec 1개 / eval pipeline 1개 강제.
- "선택지 제공" 이 항상 좋은 게 아님 — 일관성 우선.

### 패턴 7: 위반 단계적 조치
콘텐츠 수정 요청 → 미노출 → 등록 제한. **3 단계 escalation.**

→ npc-eval / welfare-gate 위반 처리:
1. 작가 알림
2. NPC 응답 일시 차단
3. 캐릭터 archive

## ★ 핵심 인사이트

### 1. ★★★ §5 생성형 AI = NPC 게임 직접 의무
한국 법령상 AI 결과물 *고지 + 표시* 는 강제. NPC 게임이 한국 출시면 즉시 적용 의무.

### 2. negative list + open clause 의 정당성
"무엇이 안 되는가" 8 + "그 외 정책 위반" — 룰 폭증 없이 미래 사례 흡수.
→ BEHAVIOR.md anti-pattern 도 동일 형식 강제 가치.

### 3. 합법 ≠ 허용
플랫폼 기준 > 법적 기준. AI welfare 도 동일: 법 없어도 우리는 강제.

### 4. "완결성" = 신뢰의 근간
외부 의존 차단 = 사용자가 *예측 가능한* 경험. 6번째 토스 가이드 중 가장 강한 표현.
→ NPC "대화 내 완결" 룰 추가.

### 5. ★ 토스 7 가이드 통합
| # | 가이드 | KPI 표면 |
|---|------|------|
| 1 | 브랜딩 | 혼동 방지 |
| 2 | 다크패턴 | 자율성 |
| 3 | UX 라이팅 | 친근/예측 |
| 4 | 해상도 | 일관 렌더 |
| 5 | 개요 | 콘텐츠/chrome 분업 |
| 6 | 프로세스 | lifecycle 예측 |
| 7 | **정책** | **negative space + 사용자 보호 + 완결성** |

→ 7번째 표면 = *경계 정의*. 다른 가이드가 *어떻게* 라면 정책은 *무엇이 불가*.
→ "예측 가능성 + 콘텐츠 우선 + 사용자 보호" 3축 굳어짐.

## 즉시 적용 후보

### 1. ★★★ 게임 출시 전 AI 고지/표시 의무 명시
ai-npc-game 프로젝트에 *법률 axis* 추가. 한국 출시 전 법률 검토 의무 명시.

### 2. ★★★ welfare-gate framing 보강 (3 축 → 4 축)
현재: welfare + 사용자 보호 + 작가성. 추가: **법적 고지·표시 의무** axis.

### 3. ★★ negative list + open clause 형식
BEHAVIOR.md anti-pattern 섹션 형식 표준화.

### 4. ★ "완결성" 룰
NPC 응답이 대화 내 완결 — 메타 회피/외부 의존 금지.

| 차원 | 1~5 | 6 (프로세스) | 7 (정책) |
|------|---|---|---|
| 축 | 공간/형태 | 시간 | 경계 |

| Sub brain 가장 강한 차용 | 룰/어휘 | 4 axes / 양방향 | **AI 고지·표시 / 4번째 윤리 축** |

→ 정책 가이드 = 토스 7종 중 **NPC harness 에 가장 직접적**. 한국 법령 반영 부분이 게임 출시 의무.

## 제한·주의

- 한국 법령 종속 — 글로벌 NPC 게임 출시 시 별도 검토.
- 토스 단일화 (로그인/결제/광고) 는 토스 자산 기반 — 우리는 직접 사용 불가.
- 8개 금지 카테고리는 NPC 게임과 직접 무관 (단, 사행성·도박 NPC 행동은 검토).
- 의료/금융 규제는 우리 도메인 외.
