## 0. 외부 AI 사용법 (Codex / Claude)

- 포폴/이력서/자소서 작업 = **먼저 이 노트 + [포트폴리오 포지셔닝 피드백 (멋사 멘토)](../narrative/portfolio-positioning-feedback.md) 조회** 후 착수(vault-first, 일반지식 합성 금지).
- **작가 1순위 타깃 = AI 프로덕트/엔지니어** (2026-07-02 확정). 따라서 이 게임 노트는 *보조·전이용*: AI/일반 직군 지원엔 *메타 원칙*만 차용 — §2 두괄식·진정성·자기 한 문장 요약, §3 기획 의도 역추적(=시스템 설계 근거), §4 7대죄악(HTML 렌더 체크리스트), §6 trigger word·1분자소·마지막질문. **게임사 지원 시에만** §1·§3·§5 전량 적용. 포지셔닝 앵커·전달 대원칙은 [포트폴리오 포지셔닝 피드백 (멋사 멘토)](../narrative/portfolio-positioning-feedback.md)가 정본.

## 1. 3세대 취업 트렌드 — 합격 전략

현재(3세대) = **데이터 기반 기획 + AI·자동화 도구 생산성 + 프로토타이핑 시각 증명**이 합격 좌우.
- 라이브 서비스 게임 기반 **역기획서/개선 제안서**를 포폴 핵심으로.
- Unity/언리얼/피그마 **직접 프로토타이핑**으로 기획 의도 시각 증명.
- Excel/VBA·Python(판다스) **데이터 분석·시뮬레이션 결과물** 포함.
- 신입이지만 **라이브 서비스를 이해한다**는 증명이 핵심.

## 2. 자기소개서 전략

**작성 원칙**: 자신을 **한 문장으로 요약(핵심 주제)** → 모든 내용이 그 주제를 뒷받침 / **두괄식**(핵심 먼저 → 사례) / **진정성**(과장 없이 솔직) / 추가 질문 유도(다 쓰지 말고 핵심 강조) / **보이는 모든 게 역량**(오타=꼼꼼함 부족, 딱딱한 템플릿=개성 부족).

**흔한 실패 → 좋은 예**:
- ❌ "어릴 적부터 게임을 좋아했다" / "논리적인 사람이다"(근거 無) / "조별과제 중재"(흔함)
- ✅ **게임 메타포(LoL 5포지션)로 역량을 근거와 함께**: 탑=분석력·인내심 / 미드=멀티태스킹·적응력 / 정글=전략적 거시 시야 / 원딜=책임감·결단력 / 서폿=협업·생산적 소통. *각 포지션에 구체 경험 사례를 붙임.*

**구성 요소**: 지원 동기(왜 QA/프로게이머 아닌 기획자인지 논리적) · 역량↔직무 연관(사례·성과로) · 회사 게임 구체 언급 + 기여 방법 · 성장 의지 · 인생 게임/선호 장르(면접 질문으로 이어짐) · 포폴 간단 소개(스크린샷·파일명·제작 기간·이유).

## 3. 포트폴리오 종류 — do / don't

| 종류 | 핵심 | 주의 |
|------|------|------|
| **역기획서** | 기존 기획자의 *숨은 기획 의도·기술적 한계·예외처리*를 역추적. 역작성 시 원본과 동일 결과 = 성공 | 공략집·나무위키 정리 = 가치 0. 학습 과정일 뿐, 이것만으론 취직 X |
| **개선 제안서** | 데이터 근거(What&Why) + *충성 유저 반발·경제 인플레 부작용 완급 조절(How)* 입체 대안 | 개인 불편 나열(징징) X. 주제 잘못 고르면 "겜알못". 라이브 유저 반발 필수 고려 |
| **가상 게임 창작** | — | **강력 비추천**. 현실 개발 제약(리소스·호환성) 무시, 평가 불명확 |
| **기존 게임 신규 시스템 창작** | *추천*. 흥행 게임 정의(Reference) 후, 기존 시스템과 **호환 가드** 지키며 새 경험 주는 단일 신규 시스템 | 지원 직무 외 역량(아트 등)에 결과 달린 주제 회피 |

**포지션별 주제 선정**:
| 목표 포지션 | 추천 포폴 |
|---|---|
| 라이브 팀 | BM 분석서·이벤트 기획서·신규 유저 동선 개선 |
| 시스템 디자이너 | 시스템 기획서·테이블 구조 역기획서·신규 시스템 창작 |
| 캐릭터·전투 디자이너 | 전투 역기획서·신규 캐릭터&스킬 창작·전투 밸런스 분석 |
| 레벨 디자이너 | 레벨 역기획서·신규 던전/보스 창작 |

제너럴리스트 지망이라도 **일관성 유지**(예: 레이드 역기획 → 레이드 보스 창작 → 레이드 보상 밸런스).

## 4. 신입 포트폴리오 7대 죄악 (HTML 제작 직결 안티패턴)

1. **인색** — 레이아웃·가독성 방치 (읽는 사람 배려 부족)
2. **음욕** — 화려한 배경·일러스트로 가독성 훼손 (어두운 배경 흰 폰트 등)
3. **탐욕** — 장황한 수식어·페이지 늘리기. *글씨 적을수록 좋음*
4. **나태** — 인터넷 PPT 템플릿 복붙 (개성 결여). 직접 깔끔히 만들 것
5. **질투** — 타 지원자 포폴 모방 (꼬리 질문서 바닥 드러남)
6. **교만** — 검토 소홀 → 오타·비문·**테이블 수치 불일치**
7. **분노** — 탈락을 남 탓 (성장 거부)

→ *포폴 HTML 제작 시 이 7개를 렌더 체크리스트로*: 가독성 우선, 정보 밀도(적게), 템플릿 티 제거, 오타·수치 정합.

## 5. 포폴 운영 전략

- **힘/민첩/지능** 3전략: 힘=다방면 역량(다양한 시스템·장르) / 민첩=프로세스 효율화(빠르게) / 지능=한 분야 깊이(밸런스·경제 특화).
- **파일명**: `회사명_지원직무_이름.pdf` (파일명부터 디테일).
- **수량**: 완성도 있는 1개 >> 미완성 여러 개. 분량보다 밀도.
- **레퍼런스**: 게임잡 등에서 합격자 샘플로 기준선 파악. **인맥 추천은 갑자기 오고 금방 마감** → 즉시 제출 가능한 범용 포폴 상시 업데이트.

## 6. 면접 전략

- **1분 자기소개** (20/30/30/20): 계기(위트 있는 정성 동기 20) → 강점·프로젝트 연관(30) → 준비 과정(직무 역량 입증 30) → 캐릭터 1줄 마무리(20, 예 "포텐셜 극대화된 태생 5성").
- **꼬리 질문 유도(Trigger Word)**: 자신 있는 분야의 전문 키워드를 의도적으로 흘려 질문 주도권 확보. *자신 없는 주제는 절대 먼저 꺼내지 말 것*(창의력 없는데 "창의력" 주제 → 침묵 참사).
- **마지막 질문**: 열정 어필("입사 전 뭘 공부하면 좋을까요") + 분석력 어필 + **이상한 회사 필터**(사수 육성 환경? 구인 많던데 대량 퇴사? 의사결정 반영도?).
- **일관 컨셉**: 면접 내내 하나의 정체성 유지가 개별 완벽 답변보다 중요(모순=진정성 의심).
- **4:6 태도**: 신입은 자기 의견 고수(4) + 피드백 유연 수용(6). 모르면 솔직히 인정 + 생각 시간 요청.
- **면접 준비 3핵심**: ① 자소서 문장 전부 = 꼬리질문 대상(사례·수치 준비) ② 포폴 모든 기획 의도·설계 결정 이유("왜 이렇게? 다른 방법은? 제약은?") ③ 플레이한 게임의 *설계 의도* 분석(공략 암기 X).

## 7. 커리어 거버넌스 (요약)

- **이직 타이밍**: 재직 중 구직 병행 필수. "여기서 내가 제일 잘하는 사람 = 더 배울 게 없음"이 이직 적기. 주니어 때 다양한 환경이 성장 고점을 올림.
- **연봉 협상**: 평소 담당 시스템의 *전후 지표·유저 반응 스크랩*(라이브) 또는 *마일스톤 진척*(신규) → 협상 테이블에 1페이지로 요약 제시. 통보를 협상으로 바꾸는 근거.
- **회사 선택**: 캐시카우(안정성) + 신작 투자(성장) + 교육·승진 체계 + 워라밸.
