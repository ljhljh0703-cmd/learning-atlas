# 이미지 프롬프트 컴파일러 규칙 — gpt-image-2 긍정형 제약 체계

*출처: `kimsh-1/gongnyang-prompt-kit` (commit `e9802a5`). Codex 흡수 → ③Gate.*

> **설치 보류 명시**: 원본은 검증기(`check_prompt.mjs`) 포함 풀 스킬이나, **풀 스킬 설치는 작가 컨펌 전까지 보류**한다(작가 현재 게임 스프라이트는 pixel-art-pipeline 사용 중 — 다른 도구). 이 노트는 **이전 가능한 규칙(RULES)만** 캡처한다. 검증기 결과(`14/14 fixtures green`, `2048x2048` 한글 반복 만점 등)는 전부 **source-reported (미검증)** — 이번 세션에서 이미지 재생성 안 함.

## 0. 왜 규칙만 흡수하는가

원본 도구는 gpt-image-2 전용 검증기까지 갖춘 프롬프트 컴파일러다. 하지만 그 규칙 알맹이는 도구와 무관하게 **HTML 히어로 이미지 · README 다이어그램 · 카드뉴스 · 포트폴리오 비주얼** 제작에 폭넓게 유효하다(infra-0 원칙: 설치보다 룰셋 추출 재구성). 아래 8종이 그 알맹이다.

## 1. 이전 가능한 규칙 8종

1. **긍정형 제약이 기본값** — `no crowd` 같은 부정문은 모델이 오히려 crowd 를 읽어 렌더할 수 있다. 장면 배제는 원하는 상태로 재서술한다. 예: `no crowd` → "프레임 안엔 인물 한 명, 깨끗한 단색 배경". `Negative:` 섹션 자체를 금지.

2. **SD 시대 품질태그를 버린다** — `8k`, `masterpiece`, `best quality`, `award-winning`, `beautiful` 류는 품질 신호가 아니라 노이즈. 기준이 프롬프트 밖에 있으면 모델은 평균값을 낸다. 빈 형용사 → 수치·팔레트·몸 반응으로 환원. 예: "professional, 8k, beautiful poster" → "상단 1/3 타이틀 밴드, 여백 60%, 메인/보조/포인트 HEX 60·30·10, 무광 종이 질감".

3. **카메라 스펙 → 장비명이 아니라 보이는 결과** — `Canon R5`·렌즈명·조명 장비명 대신 "얕은 심도", "배경이 부드럽게 떨어짐", "긴 소프트 섀도", `key:fill 1:2` 같은 결과 토큰을 쓴다. 렌즈=압축감/심도/분리, 조명=방향/hard·soft/그림자/비율, 색=HEX 3~5개+색온도, 재질=빛 반응 방식.

4. **Format A vs Format B — 섞지 않는다**
   - **Format A**(라벨 6섹션): 구조 있는 산출물(포스터·키아트·인포그래픽·도감·카드뉴스·만화). `Scene / Camera / Lighting / Color grading / Texture·Medium / Text-in-image / AR`. 라벨 섹션으로 attention 을 분산 제어.
   - **Format B**(콤마 평탄 단문): 단독 인물 화보. 슬롯 순서 고정 → 대량 변주·diff 용이.
   - 두 포맷을 혼합하지 않는다.

5. **카테고리 라우팅 C1~C12** — "이미지 프롬프트"라는 큰 범주로 처리하지 않는다. 패션·화보/뷰티/한국어 포스터/제품 도감/캠페인/인포그래픽/카드뉴스/브랜딩 목업/3D 아이콘/만화/시네마틱 키아트/프레젠테이션 덱 등으로 라우팅. 컷타입·기본 AR·필수 디테일·텍스트 정책이 카테고리마다 다르다. 카테고리 확정 후 필요한 레퍼런스만 읽는다(코어/레퍼런스 분리).

6. **텍스트 렌더 = 영역 문법 + 롤 라벨 + 따옴표 문자열** — 한글 카피는 "위에 글자"라고 감으로 쓰지 않는다. 3x3 위치·상단/하단 밴드·safe area 로 영역을 고정하고, headline/subhead/callout/caption 롤 라벨을 붙인다. 카피는 **따옴표로 고정**, KO/EN 은 한 따옴표 안에 섞지 않음, 플레이스홀더는 그대로 렌더되므로 금지, 한글 스펠아웃 금지(하이픈이 렌더될 수 있음). 실측 교훈(source-reported): 텍스트 정확도의 1순위 레버는 quality 가 아니라 **캔버스 크기**.

7. **컨셉 변주는 한 번에 축 하나만** — 사조·몸 반응·모순쌍·컬러 번역·타이포 아트를 축으로 두되, 두 축을 동시에 흔들면 무엇이 결과를 바꿨는지 추적 불가. `같은 피사체 + 같은 구도 + 같은 조명 + M축만 변경` → 살아남은 축 값 하나를 골라 다음 축 미세 조정.

8. **룩 프리셋은 섞지 않는다** — 룩은 조명·팔레트·질감·구도를 묶은 드롭인 패키지. 프리셋을 혼합하면 팔레트·질감이 충돌해 룩이 죽는다. 브랜드 팔레트가 있으면 HEX 만 교체하고 구조는 유지. 첫 사용 시 2컷 변주로 검증 후 확정.

**(+보너스) 검증기 = 문서화된 취향의 집행자** — 좋은 프롬프트 시스템은 예시만 많으면 부족하다. 금지어·포맷·크기·텍스트 정책을 검증기로 기계화해 집행한다(AR 끝 토큰 누락, 슬롯 토큰 잔존, 화이트리스트 밖 부정문, SD-era 폐기어, Midjourney 플래그, 텍스트 중복, 티어 미선언 네거티브 등을 잡음). 풀 스킬 도입 시에만 이식 대상.

## 2. 적용 후보 (보류 — 컨펌 후)

- **포트폴리오/README 이미지**: 구조도는 규칙5(C6/C12)로 노드·연결·방향 좌표화, before/after 는 규칙7(축 하나만), 다이어그램 텍스트는 규칙6(핵심 라벨만 따옴표 고정).
- **HTML 외부 공유 페이지**: 히어로 이미지는 룩 프리셋 1개, 기술 설명 이미지는 복잡 도표 문장 좌표화.
- **사용자 "예쁘게/고급스럽게" 요청 처리**: 그대로 쓰지 않고 → 카테고리 결정 → 죽은 형용사 제거 → 수치/영역/팔레트/몸 반응 환원 → 포맷 A/B 선택 → 텍스트 렌더 확인.

## 3. 대조 — pixel-art-pipeline 와 다른 도구

pixel-art-pipeline 는 **게임 스프라이트**(도트·8방향·스프라이트시트) 생성 파이프라인으로 gpt-image 가 아니다. 본 노트의 규칙은 **범용 래스터 이미지**(포스터·다이어그램·히어로) 대상 — 산출물 종류·엔진 임포트 요구가 다르다. 픽셀아트 = pixel-art-pipeline, 범용 이미지 = 본 규칙으로 라우팅.

## 4. 한계

검증기는 문법·정책·포맷을 잡지만 미적 품질을 완전히 보장하지 않는다. 텍스트 렌더는 여전히 확률적 — 핵심 카피만 고정하고 보조 텍스트는 오탈자 허용/재생성 대상. 실재 인물·상표 참조 금지 원칙은 그대로 유지. Tier-2 에디토리얼 레인은 정책 의도를 명시하는 보수적 안전 레인이지 우회 기법이 아니다.
